# Проект: WorkFinder 2.0

WorkFinder 2.0 — это современный, чистый и адаптивный фронтенд-макет платформы для поиска работы, сфокусированный на отличной читаемости, минималистичном дизайне (Soft UI) и интуитивно понятной фильтрации на стороне клиента (JavaScript).

▌1. Визуальный Обзор и Стиль

▌Дизайн-философия
Дизайн основан на чистоте и контрасте. Мы используем чистый фон (--bg: #F5F7FA), скругленные углы (border-radius: 10px) и мягкие тени (--shadow: 0 6px 18px rgba(0,0,0,.08)), чтобы интерфейс выглядел современно и не перегружал пользователя.

▌Цветовая палитра
•  Основной акцент (Primary): #00bcd4 (Бирюзовый/Циан) — для кнопок, ссылок, логотипа.
•  Вторичный акцент (Accent): #ff9800 (Оранжевый) — для выделения статусов (например, Частичная занятость).
•  Фон: Светлый, чистый (близкий к белому).

▌Типографика
Используется шрифт Inter для всего сайта, обеспечивая отличную читаемость на всех устройствах.
▌2. Инструкция по Коду и Структуре Проекта

Проект состоит из трёх основных файлов, полностью готовых к запуску локально.HTML (index.html)

Назначение: Определяет структуру, подключает стили и скрипты, содержит все элементы интерфейса (хедер, поиск, фильтры, карточки вакансий).

Ключевые особенности:
1.  Семантика: Использование тегов <header>, <main>, <aside>, <footer>, <article> и role для улучшения доступности (ARIA).
2.  Дата в футере: Динамическое обновление года в футере через JS (<span id="year">).
3.  Атрибуты Data: Карточка вакансии использует data-category и data-employment для привязки к фильтрам.

### B. CSS (style.css)

Назначение: Реализует дизайн, адаптивность и стили для всех компонентов.

Ключевые особенности:
1.  CSS Variables: Все цвета, отступы и радиусы определены в :root, что позволяет менять внешний вид сайта изменением всего нескольких строк.
2.  Layout: Используется CSS Grid для двухколоночной раскладки (.job-listing-wrapper).
3.  Адаптивность (Mobile-First Approach): Медиа-запросы переключают макет на одну колонку и меняют поведение сайдбара и поиска при ширине экрана меньше 992px.
4.  Sticky Sidebar: Сайдбар с фильтрами закреплен (position: sticky; top: 18px;) для удобства прокрутки длинных списков вакансий.

### C. JavaScript (script.js)

Назначение: Обеспечивает динамическую фильтрацию и интерактивность без перезагрузки страницы.

Ключевые особенности:
1.  Функция applyFilters(): Центральный хаб логики. Она считывает активные состояния фильтров и значения поисковых полей, а затем скрывает/показывает карточки вакансий (card.style.display = 'none').
2.  Реактивность: Фильтры и поля поиска срабатывают не только по клику на "Найти", но и по событию input (Live Search).
3.  Обработка кликов: Реализовано переключение класса .active на кнопках фильтров.
4.  Демо-действие: Кнопка "Откликнуться" вызывает alert с информацией о вакансии.

## 3. Руководство по Пользованию Сайтом

### Шаг 1: Поиск по ключевым словам и локации

Используйте верхнюю панель поиска (под заголовком "Найдите работу своей мечты"):

1.  Что ищу: Введите ключевое слово (например, Java, UI/UX, Менеджер).
2.  Где ищу: Введите город (например, Казань) или слово Удаленно.
3.  Результат: Список вакансий внизу моментально обновится.

### Шаг 2: Детальная Фильтрация (Сайдбар)

Для более точного сужения результатов используйте панель слева.

1.  Категории: Нажмите на интересующую вас сферу (например, "Маркетинг"). Сайт покажет только вакансии из этой категории.
2.  Тип занятости: Выберите желаемый формат (например, "Удаленно").
3.  Комбинирование: Фильтры работают вместе. Вы можете найти удаленную работу в сфере Дизайна.

### Шаг 3: Просмотр вакансий

Просматривайте карточки в центральной области:

•   Заголовок: Привлекает внимание (выделен бирюзовым).
•   Теги: Цветные метки (например, оранжевый для частичной занятости) мгновенно информируют о типе работы.
•   Описание: Краткое описание позволяет оценить релевантность, не покидая списка.

### Шаг 4: Отклик

1.  Если карточка вам подходит, нажмите кнопку "Откликнуться".
2.  Вам покажут всплывающее окно (в реальном проекте здесь будет форма).

## 4. План Развития и Следующие Шаги

WorkFinder 2.0 — это полностью готовый фронтенд. Для превращения его в полноценную работающую платформу необходима интеграция с бэкендом:

| Этап | Задача | Технологии |
| :--- | :--- | :--- |
| 1. Сервер и БД | Создание API для хранения, получения и обновления вакансий. | Node.js (Express) / Python (Django/Flask) + PostgreSQL/MongoDB. |
| 2. API Интеграция | Замена статических данных в HTML и JS на динамические запросы к API. | fetch() в JS для получения списка вакансий. |
| 3. Профиль Кандидата | Система регистрации, авторизации и загрузки резюме. | JWT/Сессии, хранение файлов (S3/облако). |
| 4. Профиль Работодателя| Интерфейс для размещения, редактирования и управления вакансиями.| 5. Расширенные Фильтры | Добавление фильтров по зарплатному диапазону (слайдеры) и уровню опыта. | Обновление JS-логики и добавление новых полей в HTML. |
